# v0.2 — Failover, context, cost

v0.2 turns harnessctl from a thin wrapper into something that survives a quota hit mid-task. Seven features, all opt-in, no breaking changes.

## Auto-failover

When the primary agent hits a rate limit, a token-window overflow, or an auth error, harnessctl classifies the exit reason and silently hands off to the configured fallback — no prompt, no loss of momentum.

Enable it in the agent's YAML:

```yaml
# ~/.harnessctl/agents/claude.yaml
fallback: codex
auto_failover: true
```

Generic errors still show the manual confirm prompt. Only rate/token/auth failures trigger the silent path.

## Cross-agent conversation transfer

On auto-failover, the prior user/assistant turns from the failed agent's session file are extracted, formatted as compact markdown, and prepended to the fallback agent's prompt — truncated to ~40% of the target's context window. If extraction isn't supported, falls back gracefully to the summary-only behavior.

Privacy opt-down:

```yaml
failover_transfer: "summary"   # disable transcript transfer
```

## Project context + native memory sync

One canonical preamble, mirrored everywhere each agent looks for it.

```bash
harnessctl context set "Go 1.22, postgres, follow existing patterns"
harnessctl context edit
harnessctl context clear
```

Stored at `~/.harnessctl/projects/<cwdHash>/context.md` and mirrored into `./CLAUDE.md`, `./AGENTS.md`, and `./GEMINI.md` between `<!-- harnessctl:begin/end -->` sentinels. The content outside the sentinels is left alone.

## Prompt templates

Reusable shells in `~/.harnessctl/templates/<name>.md`. `{{ARGS}}` is the only reserved token.

```bash
echo 'Review {{ARGS}} for security vulnerabilities' > ~/.harnessctl/templates/sec.md
harnessctl run --template sec "src/auth.ts"
```

## Budget guardrails

```bash
harnessctl run --budget 2.00 "refactor the payment module"
```

Scans today's run logs, sums `result.cost`, aborts if the threshold is already met. Warns at 80%.

## Cost dashboard

```bash
harnessctl stats --cost
```

Per-agent table with today, 7d, 14d spend and a 14-day sparkline.

## Replay

```bash
harnessctl logs              # find a run id
harnessctl replay <id>       # re-run with the same agent, prompt, and extra args
```

## Explicit handoff

Target a specific run for cross-agent handoff — no ambiguity, works with concurrent sessions.

```bash
harnessctl run --agent codex "refactor auth"
# → run: 1713364500000-codex  session: a3f8c012

harnessctl handoff 1713364500000-codex --agent claude "review and add tests"
```

The target agent gets a lean prompt (summary + changed files + pointer to `.harnessctl/handoffs/<run-id>.md`), not a full transcript dump. Same-agent handoff supports `--resume` (continue native session) or `--fork` (new session with context).

Shell mode also tracks sessions — after exit, harnessctl recovers session data from agent logs so `handoff` works from interactive sessions too.

## Agent comparison

Run the same prompt across multiple agents in parallel, then compare duration, cost, token usage, and output quality side by side.

```bash
harnessctl compare "fix the auth bug" --agents codex,claude,gemini
```

Each agent runs concurrently. Results are printed as a table showing status, duration, cost, token count, and a one-line summary. Every run is logged individually so you can hand off from any of them.
